# Code of Conduct

## Our Standard

We want Myosotis to be a respectful, technically serious, and welcoming open-source project.

Examples of behavior that contribute to a positive environment:

- giving constructive feedback
- assuming good intent while staying precise
- documenting tradeoffs clearly
- helping newcomers understand context
- respecting different levels of experience

Examples of unacceptable behavior:

- harassment or personal attacks
- bad-faith argumentation
- discriminatory language
- doxxing or privacy violations
- trolling, intimidation, or deliberate disruption

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, or other contributions that violate this code of conduct.

## Scope

This applies in project spaces such as:

- GitHub issues
- pull requests
- discussions
- documentation feedback
- community support channels tied to the project

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers privately through the channel listed in [SUPPORT.md](./SUPPORT.md).

## Maintainer Expectations

Maintainers are expected to enforce this code of conduct consistently and proportionally, with the goal of protecting contributors and keeping the project productive.
